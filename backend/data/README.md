# Datasets

Generated by `python -m ingest.scrape_osm` (from `backend/`).

Source: OpenStreetMap Overpass API — restaurants, cafés, pubs, bars,
fast-food and ice-cream venues inside a bounding box covering East
London (Shoreditch → Hackney → Whitechapel → parts of the City).

| File | Size | Contents | Used by |
|------|------|----------|---------|
| `seed.json` | ~163 KB | 7 anchors + 400 curated candidates | Frontend demo (set `VITE_USE_OSM=true`) |
| `seed-full.json` | ~2.4 MB | 7 anchors + 6,313 candidates (the full scrape) | Neo4j load |
| `seed-osm.cql` | ~3 MB | `MERGE` statements for the full set | `cypher-shell -f` |

## Anchor selection

Anchors are picked in two passes:

1. **Iconic-name pass** — try to match well-known East London venues
   (Beigel Bake, Dishoom, Towpath, E. Pellicci, Lahore Kebab House,
   St. John, Caravan…) in the scrape by loose name match. When the
   name is ambiguous (`St. John` could be a street, a bar, or the
   restaurant) we apply an area hint to disambiguate.
2. **Algorithmic fill** — remaining slots are filled by closest-to-
   centre, best-tag-coverage venues with distinct primary cuisines.

Each anchor gets a synthetic `searchCount`, `saves`, and `directions`
count seeded by its id, so the demo profile-card numbers are stable
across re-runs.

## Demo-quality filter

The `seed.json` subset keeps only venues that:
- Have an explicit `cuisine` OSM tag (not the generic `misc` fallback)
- Have at least 2 derived vibe items
- Are within ~3.5 km of the user centre
- Sorted by `(distance, -tag_coverage)`, capped at 400

This keeps the map readable at zoom 13–14 without dropping the rich
storytelling venues.

## Loading into Neo4j

```bash
cypher-shell -a "$NEO4J_URI" -u "$NEO4J_USER" -p "$NEO4J_PASSWORD" \
  -f cypher/schema.cql                          # constraints + indexes
cypher-shell -a "$NEO4J_URI" -u "$NEO4J_USER" -p "$NEO4J_PASSWORD" \
  -f data/seed-osm.cql                          # all 6,300+ venues
```

Or, if you only want the small demo set:

```bash
cypher-shell -f cypher/schema.cql
cypher-shell -f cypher/seed.cql                 # 22-venue hand-curated set
```

## Re-scraping

```bash
cd backend
python -m ingest.scrape_osm
```

Takes about 30-60 seconds for the Overpass query. Reuses the same
neighborhood centroids and dish/cuisine/vibe derivation rules.
